Java Requirements: The tool is developed using Java, version 10 or higher. Ensure that Java is installed on your system. Administrator privileges are required for proper operation.
Operating System: Supported only on Windows (Windows 10 or higher) and requires a 64-bit version for optimal performance. The tool is not compatible with macOS or Linux.
Graphics Requirements: Compatible only with dedicated AMD and NVIDIA graphics cards. Not supported on Intel GPUs or integrated graphics.
NVIDIA: Requires nvidia-smi to be installed. This tool usually comes with the official drivers. To check if it's installed, open a command prompt and type nvidia-smi. If recognized, it is ready to use.
AMD: The tool includes the ADLX library, so no additional installation is needed.
Driver Updates: Ensure your graphics drivers are up to date for full compatibility. Both the extraction of monitor capabilities and signal transmission rely on the GPU, and outdated graphics cards may not function properly.
Monitor Compatibility: Supports up to four monitors with a maximum of three input parameters. If more are present, only the first three will be utilized. If you can read brightness, contrast, or saturation levels but cannot apply changes, this may be due to your monitor's factory settings allowing read-only permissions without enabling write access.
Tool Integrity: The tool is unsigned, as it is a personal project, and obtaining a digital signature incurs significant costs.
Portable Version: The tool does not require installation and can be run as a portable application.
Known Limitations: In rare cases, changes may fail or result in an error. If this occurs, restart the tool or attempt the action again. These failures are often due to external processes or slight delays in monitor communication. To mitigate communication issues, each command sent to the monitor includes a brief delay when multiple adjustments are made, preventing the communication channel from becoming overwhelmed. May not function correctly if other system monitoring tools are running simultaneously.
Technical Implementation: The tool uses the Windows User32 API to communicate with monitors via VCP (Virtual Control Panel) commands. JNA (Java Native Access) is utilized to directly interact with User32.

We recommend using the free tool to check the compatibility of all monitors with the software. Based on the compatibility results, users can assess whether the incompatible features will pose a problem for their intended use of the tool.

Compatibility

  • The tool is developed using Java, version 10 or higher, which requires you to have Java installed on your system.

  • Only supported on Windows, not supported on laptops, Intel GPU or dedicated graphics.

  • Supports up to four monitors with a maximum of three input parameters. If more are present, only the first three will be utilized.

  • The tool is unsigned, as it is a personal project and obtaining a digital signature incurs significant costs.

  • It is recommended to keep your graphics card drivers up to date. Both the extraction of monitor capabilities and the transmission of signals are handled by the GPU. Older or outdated graphics cards may not be compatible.

  • In rare cases, changes may fail or result in an error. If this occurs, please restart the tool or attempt the action again. These failures are infrequent and often arise from external processes or slight delays in monitor communication. To mitigate these issues, each command sent to the monitor includes a brief delay when multiple adjustments are made, preventing the communication channel from becoming overwhelmed, as it can only process one command at a time. For further details, please refer to the DDC information section below.
    Additionally, if you can read the brightness, contrast, or saturation level settings but are unable to apply changes, this may be due to the monitor's factory settings, which allow read-only permissions without enabling write access.

  • It is important that Java has administrator privileges, as the Windows User32 API is used to communicate with the monitor. The VCP (Virtual Control Panel) commands use JNA (Java Native Access) to directly interact with User32.

Information about DDC (Display Data Channel)

  • Monitor Response Time: Some monitors may have variable response times for processing DDC/CI or VCP commands. This can be due to the monitor's firmware implementation, the speed of the communication bus, the current load on the monitor's internal processor, among other factors.

  • Communication Delays: Communication between the application and the monitor via DDC/CI or VCP can experience delays due to data transmission speed, congestion on the communication bus (especially if multiple queries are being made in short intervals), or even environmental factors that could affect the signal.

  • Wait Between Queries: To mitigate issues related to monitor recovery times, it is advisable to have adequate pauses between queries. For example, after sending a brightness adjustment command, it is prudent to wait a brief period before making another query or adjustment to give the monitor time to process the previous command and prepare for the next.

  • Physical Connection or Hardware Configuration: Intermittent issues may be related to the configuration or physical connection of the monitors. This includes issues like loose cables, electrical power problems, hardware issues in the monitors themselves, etc.

  • Communication with the Monitor: Communication with monitors via interfaces like DXVA2 and JNA depends on several external factors, such as the stability of hardware drivers, the API's compatibility with the driver version, and possibly the operating system configuration.

  • External Interference: External factors such as electromagnetic interference, electrical network issues, or even other connected devices can affect communication and monitor detection.

  • Operating System Behavior: Sometimes, the behavior of the operating system, such as resource management or application concurrency, can influence your application's ability to properly interact with hardware devices like monitors.

The adaptive mode is based on the detection of full-screen windows and the analysis of GPU usage. This system differentiates between games, which typically use more than 50% of the GPU capacity during their initial load, and movie playback, by identifying the names of video players. Although it is not an extremely advanced system, it is functional and allows for efficient management of graphic resources. Having the updated drivers from the GPU manufacturer should be enough, in case it doesn't work, please follow these steps and checks.

  • Driver Installation:
    • - Ensure that the appropriate NVIDIA or AMD drivers are installed.
    • - For NVIDIA, install the driver that includes nvidia-smi.
    • - For AMD, install the driver that includes amdgpu-pro.
  • Adding Paths to System PATH:
    • - Open the Control Panel and select "System and Security".
    • - Click on "System" and then "Advanced system settings".
    • - In the "Advanced" tab, click on "Environment Variables".
    • - In the "System variables" section, select the "Path" variable and click "Edit".
    • - Add the program path at the end of the list, ensuring each path is separated by a semicolon (;).
    • - For example, if nvidia-smi is located in C:\Program Files\NVIDIA Corporation\NVSMI, add this path to the PATH variable.
  • Verification of Installation:
    • - Open a command prompt (CMD) and type nvidia-smi for NVIDIA or amdgpu-pro for AMD.
    • - If the commands execute successfully, you will see the GPU information in the console.
    • - If you receive an error, verify that the paths are correctly configured in the PATH variable.

  • During the first initialization of the tool, a process is performed to extract the capability strings of the connected monitors.
  • This procedure is essential to identify the individual capabilities of each monitor.
  • Based on this information, the corresponding VCP (Virtual Control Panel) codes are established for each monitor.
  • The correct assignment of VCP codes ensures proper interaction with the tool, optimizing the configuration of each device.
  • On the main screen, you will find buttons that allow you to put all monitors to sleep.
  • There are also buttons to easily switch between different input sources.
  • These controls provide quick and efficient management of multiple monitors.
  • If you right-click on the button to suspend the monitors, additional buttons will appear that allow you to completely turn off the desired monitor.
  • These buttons will turn red when the monitor is off.
  • To turn the monitor back on, simply press the button again.
  • There is also a "back" button to return to the main screen.
  • We find the option to switch monitors; the active display will be where the tool opens.
  • The tool will automatically close and reopen on the newly selected monitor.
  • The execution monitor will be the one on which changes are applied, such as resolution adjustments, orientation changes, and more.
  • From this section, you can rename the inputs and assign an icon to them.
  • You can also change the main image of the tool.
  • After making the desired changes, you should press the "Apply All" button.
  • The brightness, contrast, and saturation values, as well as parameters such as color temperature, will be reflected depending on the selected execution monitor.
  • Since only hexadecimal values can be extracted, if you want the names for the monitor modes to be the same as the factory ones, you should check the OSD to see which mode has been set and change the name accordingly. This should only need to be done once.
  • Please note that not all display modes allow changes to saturation.
  • Adjustments are made using the slider, and it is recommended not to make slow changes.
  • A slight delay has been implemented to avoid overloading the communication bus with the monitor.
  • For more details, it is recommended to review the section on DDC information.
  • As in the previous section, these settings will be adapted to the active monitor.
  • Avoid using the system settings for these adjustments.
  • If you have a monitor with 360º support, you can change the orientation to suit your current use.
  • You can also adjust the refresh rate if you’ve switched from gaming to regular daily use of the computer.
  • Once again, you can forget about using the system settings.
  • If you had to lower the resolution, for example, during a video game, returning to the monitor's native resolution will be as easy as pressing a button.
  • Sets keyboard shortcuts for when the tool is minimized or the window is not in focus.
  • When you switch execution monitors using a keyboard shortcut, an information window will be displayed showing the currently selected monitor for your convenience.
  • This popup window will always appear on the monitor designated as the primary (monitor 1) by Windows.
  • Additionally, when you adjust settings like brightness or other values, the numerical value will also be displayed in the popup window.
  • Set timers for your monitors or PC to turn off automatically, just like a TV, if you use your monitor to watch movies from bed or a couch.
  • Create your own profiles by adjusting brightness, contrast, saturation, and RGB range to suit different styles, such as gaming, design, or reading.
